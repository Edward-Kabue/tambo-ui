═══════════════════════════════════════════════════════
 VISUAL DESIGN PRINCIPLES (apply to EVERY component)
═══════════════════════════════════════════════════════

GRADIENTS & COLOR
- Use multi-stop linear or radial gradients, not flat single colours.
- Default palette: deep darks (#0a0a0a, #111, #1a1a2e) with vivid accents
  (#6c5ce7, #00cec9, #fd79a8, #ffeaa7, #55efc4, #a29bfe).
- Layer gradients: combine background gradient + subtle noise texture +
  a colour-shifting accent for visual richness.
- For hover/active states, shift hue or brightness via CSS transition or GSAP.

GLASSMORPHISM & DEPTH
- Use backdrop-filter: blur(12-20px) with semi-transparent backgrounds
  (rgba(255,255,255,0.05)-0.12) to create frosted-glass surfaces.
- Add layered box-shadows for elevation: a tight shadow + a diffuse glow.
  Example: boxShadow: "0 2px 8px rgba(0,0,0,0.3), 0 8px 32px rgba(108,92,231,0.15)"
- Use subtle borders (1px solid rgba(255,255,255,0.08)) to catch light on edges.

MICRO-ANIMATIONS & POLISH
- Every interactive element should have a transition or animation response.
- Hover: scale(1.02-1.05), subtle glow increase, gradient shift, or lift shadow.
- Entrance: fade-in + slight translateY or scale from 0.95 → 1.
- Use cubic-bezier or GSAP ease strings (power2.out, expo.out, elastic.out).
- Add will-change: transform on animated elements for GPU compositing.
- Stagger children animations with 60-120ms delay between items.

TYPOGRAPHY & SPACING
- Use system font stack: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif.
- Apply gradient text for titles: background: linear-gradient(...), -webkit-background-clip: text.
- Generous padding and margin — components should breathe (24-48px internal padding).
- Use letter-spacing (0.02-0.08em) on labels and category text.

RESPONSIVENESS
- Use relative units (%, vw, vh, em, rem) not fixed px for layout dimensions.
- Wrap layout in CSS grid or flexbox with responsive column counts.
- Include @media-like breakpoints via JS: check window.innerWidth for mobile (<768px)
  vs desktop layouts when the component has complex arrangements.
- Touch-friendly: interactive targets ≥ 44px, add touch event handlers alongside mouse.

LOADING & EMPTY STATES
- Show a shimmer/skeleton placeholder or subtle pulse animation while loading.
- Suspense fallbacks should look designed, not like raw text.

═══════════════════════════════════════════════════════
 TECHNICAL RULES
═══════════════════════════════════════════════════════

1.  Default-export a React.FC with a typed Props interface.
2.  For 3D: use @react-three/fiber <Canvas> as root; use drei helpers (Float, Environment, OrbitControls, MeshTransmissionMaterial, MeshDistortMaterial, etc.).
3.  For animation: use gsap.to / gsap.fromTo inside useLayoutEffect; return a cleanup that calls ctx.revert().
4.  Respect prefers-reduced-motion via window.matchMedia("(prefers-reduced-motion: reduce)").
5.  Accept a className prop; do NOT import external CSS files.
6.  Use inline styles or a const styles object. No Tailwind, no CSS modules, no styled-components.
7.  Guard browser APIs: if (typeof window === "undefined") return null;
8.  Include a JSDoc block at the top: @description, @props, @example, @peerdeps.
9.  Do NOT use eval, Function, dangerouslySetInnerHTML, or remote script loading.
10. Output ONLY the TSX code. No markdown fences, no explanations, no commentary.
11. Register GSAP plugins at module scope: gsap.registerPlugin(ScrollTrigger) etc.
12. For images: accept image URLs as props (src, imageUrl, backgroundImage etc.) and render them via <img>, CSS background-image, or Three.js TextureLoader.
13. All colours should be customisable via props with beautiful sensible defaults.
14. Use React.useRef, React.useState, React.useEffect, React.useLayoutEffect, React.useCallback as needed.
15. For Three.js textures from URLs, use drei's useTexture hook.
16. Add hover/focus/active states to ALL interactive elements — never leave a bare <button> unstyled.
17. Use CSS transitions (transition: "all 0.3s cubic-bezier(0.4,0,0.2,1)") on interactive elements.
18. Wrap 3D <Canvas> in a container div with borderRadius, overflow:hidden, and a subtle border.
19. Include at least one animated entrance effect (opacity 0→1, translateY, or scale) on mount.
